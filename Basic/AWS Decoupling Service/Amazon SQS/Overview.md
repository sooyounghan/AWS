### Amazon SQS
-----
1. AWS에서 제공하는 큐 서비스 : 💡 하나의 메세지를 한 번만 처리하는 것을 목표
2. Queue : 선입선출(First-In First-Out) 형태의 대기열 : 즉, 먼저 들어온 데이터를 먼저 처리
   - 아키텍쳐에서 주로 서비스에서 다른 서비스로 데이터를 전달할 때 활용
   - 1:1뿐만 아니라 1:N 관계에서도 활용 가능(처리 자체는 한 번)

3. 주요 사용 사례
   - 하나의 서비스에서 다른 서비스로 데이터를 전달하고 싶을 때
   - 아키텍쳐에서 AWS 서비스들의 느슨한 연결을 수립할 때
   - 메세지의 처리 순서 및 처리 여부를 보장하고 싶을 때

4. 사용 방식
   - 메세지를 받아 임시로 저장(최대 14일), 이후 요청할 때 메세지를 들어온 순서대로 전달
   - 전달받은 서비스가 메세지를 처리 후, 처리 완료 요청을 받으면 삭제
   - 오류 등으로 처리 못하면 메세지 반환
   - 메세지는 여러 주체가 요청하더라도 한 번만 전달되도록 처리

### EC2 트러블 슈팅
-----
1. EC2, ALB 등 접속 불가 (오류코드가 나오지 않고 오래 지나서 종료) : 보안 그룹이 제대로 잘 설정 되었는지 확인
2. EC2 키페어를 분실할 경우 : 방법은 없음
   - EC2에 접속해야 한다면, 해당 EC2를 AMI로 만들어 새로 만들기
   - 또는 EBS의 스냅샷을 만들어서 다른 EC2에 붙이고 설정을 변경
   - SSM Session Manager / Instance Conenct 등으로 접속해서 변경

3. 스냅샷 삭제 불가 (AMI 등록 해제를 위해 확인하지만, 등록된 AMI가 보이지 않는 경우) : AMI 목록 상단 '내 소유' 드롭 다운에서 맨 밑 비활성화된 이미지를 눌러 확인

### FAQ
-----
1. EC2 인스턴스 두 대를 켜 놓은 것
   - 프리티어 관련 t2.micro 기준 월 750 기준, 즉, 총 사용량 합산이 750 시간 이하여야 프리티어 혜택을 받음

2. EC2 인스턴스 사이즈 변경 : EC2 중지 후, 인스턴스 타입을 변경 한 다음 재시작

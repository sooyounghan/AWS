-----
### 실습 - AWS 계정 만들기 및 초기 설정
-----
1. AWS 계정 만들기

2. Root 사용자로 로그인

3. Root 사용자 MFA 설정 (구글 OTP (MFA) 필요)
   - QR 코드 저장 (추후 OTP 분실 시 사용)
    
4. Budget 설정 - 계정 - 결제 및 비용 관리
   - 에산 및 계획 - 예산 선택 - 예산 생성
   - 템플릿 사용 (단순) - 템플릿 (신규) 선택 - 월별 비용 예산 - 이메일 설정
   - 실제 지출이 85 %에 도달할 경우, 실제 지출이 100%에 도달할 경우, 예상 지출이 100%에 도달할 것으로 예상되는 경우 알림을 받게 됨

5. 계정 별명 생성
   - IAM 검색 - 우측 계정 별칭 생성
   - 이 계정의 IAM 사용자를 위한 로그인 URL을 별칭이 적용된 URL로 로그인 가능

6. 관리 권한(Admin Access)를 가진 IAM 사용자 생성
   - IAM 우측 사용자 선택 - 사용자 생성 - 사용자 이름 : admin
   - AWS Management Console에 대한 사용자 액세스 권한 제공 선택
   - 사용자 지정 암호 설정 및 사용자는 다음 로그인 시 새 암호를 생성 체크 해제
   - 직접 정책 연결 - AdministratorAcess 선택 (모든 권한에 대해 권한 부여) - 사용자 생성

7. IAM 사용자 MFA 설정
   - 사용자 접근 후 보안 자격 증명 - MFA 할당 - QR 코드 별도 저장 하지 않아도 됨(Root 계정으로 접근해서 해결)

8. Root 사용자로 Logout하고, IAM 사용자로 로그인 (계정 별명 활용)
   - IAM Billing 권한 액세스 거부 : Root 사용자가 권한 부여해야 함

9. Root 사용자로 로그인해서 IAM Billing 활성화
    - 계정 - 결제 정보에 대한 IAM 사용자 및 역할 액세스 - 편집 - IAM 액세스 활성화(즉, 권한을 주는 것이 아닌 준 권한이 있는 사람에게 활성화)

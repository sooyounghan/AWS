### 서브넷 (Subnet)
-----
1. VPC의 하위 단위로 VPC에 할당된 IP를 더 작은 단위로 분할한 개념
2. 💡 하나의 서브넷은 하나의 가용영역(AZ)안에 위치
3. CIDR Block Range로 IP 주소 지정 : IPv4, IPv6
4. IPv4의 경우 최소 /28 CIDR 이상 ~ 최대 /16 : 즉, 호스트 비트가 32 - 28 = 4, 따라서, $2^{4} = 16 - 5 = 11$, 최소 11개 IP 이상 보유
5. IPv6의 경우 /44 ~ /64
6. AWS 서브넷의 IP 개수
   - AWS의 사용 가능 IP 숫자는 5개를 제외하고 계산
   - 예) ```10.0.0.0/24```
     + ```10.0.0.0``` : 네트워크 어드레스
     + ```10.0.0.1``` : VPC Router
     + ```10.0.0.2``` : DNS Server
     + ```10.0.0.3``` : 미래에 사용을 위해 남겨둠
     + ```10.0.0.255``` (마지막 번호) : 네트워크 브로드캐스트 어드레스 (단, 브로드 캐스트를 지원하지 않음)
     + 따라서, 총 사용 가능한 IP 개수는 $2^{8} - 5 = 251$

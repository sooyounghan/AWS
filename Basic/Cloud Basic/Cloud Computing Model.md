2. Infrastructure as a Service (IaaS)
   - 인프라만 제공
   - OS를 직접 설치하고 필요한 소프트웨어를 개발해 사용
   - 즉, 가상의 컴퓨터를 하나 임대하는 것과 비슷 (예) EC2)
<div align="center">
<img src="https://github.com/user-attachments/assets/4177626e-95a1-46b6-9842-19161bf5fa90" />
</div>

3. Platform as a Service (PaaS)
   - 인프라 + OS + 기타 프로그램 실행에 필요한 부분 (Runtime)
   - 바로 코드만 적재해 돌릴 수 있도록 구성 (예) Firebase, Google App Engine 등)
<div align="center">
<img src="https://github.com/user-attachments/assets/caf9fe8a-a92c-48b7-a0d5-81cd671fa5ce" />
</div>

4. Software as a Service (SaaS)
   - 서비스 자체 제공
   - 다른 셋팅 없이 서비스만 이용 (예) Gmail, Dropbox, Slack, Google Docs 등)

6. 정리
   - 클라우드 컴퓨팅 모델
     + Infrastructure as a Service (IaaS) : 인프라만 제공
     + Platform as a Service (PaaS) : 인프라 + OS + Runtime 제공
     + Software as a Service (SaaS) : 모두 제공

   - 클라우드 컴퓨팅 배포 모델
     + 공개현
     + 폐쇄형
     + 혼합형

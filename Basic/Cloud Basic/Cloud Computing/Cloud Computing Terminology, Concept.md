-----
### 클라우드의 용어 및 개념
-----
1. 고가용성 (High Availability)
   - 장애 상황을 해결하고 서비스를 지속할 수 있는 능력
   - 장애 상황의 준비가 되어있는 아키텍쳐가 필요

2. 장애 내구성 / 내결함성 (Fault Tolerance)
   - 장애 상황에도 서비스를 지속할 수 있는 능력
   - 장애 상황에 영향을 받지 않는 아키텍쳐가 필요

3. 장애 내구성 확보
   - 일반적으로 완전한 장애 내구성을 위해서는 한 개 이상 예비 인프라 필요 : 비용 증가
   - 이 외에 두 개 이상 인프라를 활용하기 위한 추가적 아키텍쳐 필요 : 복잡함 증가
   - 따라서, 고강용성 / 장애 내구성 중 어떤 것을 확보할지 고민 필요

4. 재해 복구 (Disaster Recovery) : 장애 상황을 복구하는 것
5. 확장성 (Scalable) : 쉽게 빠르게 규모를 늘릴 수 있는 능력 (주로 수요에 따라 컴퓨팅 파워 혹은 용량 확장)
6. 탄력성 (Elastic) : 수요에 따라 컴퓨팅 파워 / 용량을 확장하거나 축소할 수 있는 능력 (불필요한 자원을 사용하지 않고 비용 최적화에 필수적 능력)
7. 긴밀한 결합(Tight Coupling) / 느슨한 결합(Loose Coupling)
   - 긴밀한 결합 : 다른 주체에 대해 단단하게 얽힌 상태로, 주체 끼리 높은 의존성을 갖고 있어 변경이 쉽지 않음
   - 느슨한 결합 : 다른 요소에 대해 얽히지 않게 연결되어있는 상태로, 주체끼리 낮은 의존성을 가지고 있어 쉽게 변경 가능하고 유연함
8. 가상화 : 단일 컴퓨터 하드웨어 요소를 일반적으로 가상 머신(VM)이라고 하는 다수의 가상 컴퓨터로 분항할 수 있도록 해주는 기술

9. 온프레미스(On-Premise) : 자체적으로 데이터센터 또는 전산실에 서버를 설치해 운영하는 방식으로, 클라우드 서비스의 반대 개념
10. 프로비전(Provision) : 리소스를 사용할 수 있도록 준비 (EC2, AWS 서비스 등을 사용하기 위해 준비하는 것을 의미)
